What is HTML?

HTML is a hypertext markup language, used to define the structure of web pages.

HTML uses tag (or we also call them elements) and attributes inside those elements to describe the structure

How are pages loaded?

  1. Request: the client(aka your web browser + the end user) requests the resource from the server for a particular url
  2. Response: The server responds with the resources (the page and anything else it needs to build and load)
  3. Build: Browser builds the resources (interprets js, builds a dom, etc.)
  4. Render: Browser engine then renders the page for you. While each browser implements HTML5 and CSS3 standard, the implementation differs a bit from a browser to browser. Some of these can be cute, others not so much. (akin to different sql dialects)

How are pages organized?

All html pages begin with the tag 

<!DOCTYPE html>
. This denotes that this document is of html type

Further more, the pages are organized in the following way 

 
                <html>
                    <head>
    
                    </head>
                    <body>
    
                    </body>
                </html>

The Head

The head tag in html contains meta data (the data about itself) about the page. In the head tag, it contains information such as which encoding it is using and title for the page. This is also the place to include the link to external stylesheets and <style></style> tag.

The Body

This is where the content of the page goes to. Everything you see in the web browser's viewport is here.

Attributes

Attributes are fields that provides additional information about a tag. For example, the anchor tag has href attribute to describes its destination, image tag (img) has src attribute to denote the resource's location

Global attributes are attributes that are available to all elements regardless of their type. Examples of these are thing like, id, class

We use Classes and Id attributes to id a particular element or a set of elements. Id is used to uniquely identify an element, where class is used to identify all elements with that class.

DOM

DOM stands for Document Object Model, and it is a tree-like structure to describe the structure of a web page.

The Selectors

We interact with and traverse the DOM and the elements inside it using the DOM interface. It exposes selectors like document.GetElementById(), document.GetElementByClassName(), document.GetElementByTagName().. etc. To manipulate html elements using javascript

Events

Events are any type of interaction with the web page.

For example, things like click, mouse move, hover, scroll, viewport change... really about any interaction is an event

Also, there are events for any significant changes in page state, such as when an element has been loaded

Dynamic Web Design

Web page is considered dynamic, if it reacts to user interactions (the opposite of dynamic is static)

Event Handling

The browser keeps track of all the events that happen in the page. We can subscribe to that event using event listeners, such as "onclick" for handling click event for a particular element. In the event listener, we assign the event handler function to react to the particular event

The Inspector

Knowing how to use this is crucial in front end development. It's loaded with super useful functionalities that makes working with html/css/js easier and faster.
