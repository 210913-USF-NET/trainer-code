Semantic Elements

Semantic tags define structure and express the meaning of content. Semantic elements include header, footer, article, nav, etc. Non-semantic tags are tags that don't particularly describe its role in its document. Such tags include div or span.

Block vs Inline element

Block elements, such as

div
they take up the whole width of its parent elements width by default

Where as, the inline element, such as span only takes up as much space as the content needs.

Styling with HTML

While not recommended, there are certain amount of stylings you can accomplish with html tags

b tags make things bold and i tags italicize the content

You can also make the text small

s tags strikes through content and u tags underline

Note: Not only are these not the best practices, if you use these tags, you are at the mercy of each browser engine to determine how these tags look. You really don't want that. Just think Internet Explorer.
